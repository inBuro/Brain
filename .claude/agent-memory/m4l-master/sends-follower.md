## вљ пёЏ dlst dire record structure (CONFIRMED вЂ” reusable craft, see also amxd-format.md Path B)
Each field = `tag(4) + len(4 BE) + value(lenв€’8)`. dire record = `dire + 108(BE) + 100B payload`.
Payload (fixed 100 B): `type`(ln12,4B 'JSON'/'TEXT') + `fnam`(ln28, **20B value null-padded в†’ filename в‰¤19 chars + null**) + `sz32`(ln12,4B BE) + `of32`(ln12,4B BE, rel to BASE 0x20) + `vers`(ln12) + `flag`(ln12; JSON=17, TEXT=0/8) + `mdat`(ln12, checksum вЂ” Live ignores; new file can use 0).
**dires are FLAT (top-level), not nested.** Parse by walking TLV from `dlst+8` to EOF, not by trusting declared dire-len. `dlst total` field is NOT the byte length of payload (it's NГ—108+8); ignore it when parsing, recompute when writing.
**Filename в‰¤19 chars** (fnam value 20B incl. null) вЂ” `sends_version_check.js` (22) didn't fit в†’ renamed `sf_version_check.js` (19). Check name length when adding a resource.
To add an embedded file (Path B): append body after last existing embedded file; new of32 = prev_endв€’BASE; rebuild whole dlst with all 3 dire (patch JSON sz32=L1+1, shift trailing-file of32 by О”L_json, add new dire); patch ptch + mx@c; keep `0x30+mxcв€’dlst=16`.
