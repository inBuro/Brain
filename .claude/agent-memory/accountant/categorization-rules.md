# Карта категорий — метка операции Bangkok Bank → категория

Знак берём из дельты баланса (см. [bbl-statement-parsing.md](bbl-statement-parsing.md)): `amount<0` = расход, `amount>0` = доход.

## Расходы (amount < 0)
| Метка (Particulars) | Категория |
|---|---|
| `PMT FOR GOODS`, `PURCHASE E-CHN` | 🍜 Еда и магазины (оплата товаров/карта) |
| `TRF. PROMPTPAY`, `PMT. PROMPTPAY`, `TRANSFER` (исходящий), `TRF TO OTH BK` | 💸 Переводы |
| `CASH W/D ATM`, `CASH ATM W/D` | 🏧 Наличные (ATM) |
| `COM/ANNUAL FEE`, `FEE OTH BAK ATM` | 🧾 Комиссии |

## Доход (amount > 0)
| Метка | Категория |
|---|---|
| `TRF FR OTH BK`, `TRANSFER` (входящий/возврат) | 📥 Поступления |
| `BE1ST SMART` | ♻️ Кешбэк / возврат (постится задним числом) |

## Спец-правило: Аренда / Дом
Повторяющийся **фиксированный** перевод PromptPay у начала месяца (около 1-го числа; может пройти 31-м числом предыдущего). Выделяется из «Переводов» в отдельную категорию 🏠 Дом.
- **Сумму НЕ хардкодить — подтверждать у пользователя.**
- Детектор: `PROMPTPAY` с суммой == подтверждённая аренда. Проверить, что нашлось по платежу на каждый месяц; если в каком-то месяце нет — сказать пользователю (аренда могла пройти иначе, напр. через `PMT FOR GOODS`).

## Важное ограничение: у переводов НЕТ имён получателей
В выписке Bangkok Bank строки переводов содержат только тип операции (`PROMPTPAY` и т.п.), без имени/телефона получателя. Поэтому переводы конкретному человеку (напр. «Мария») **автоматически не выделить** — это самый крупный и «слепой» котёл расходов.
Решение: интерактивная пометка в дашборде (пользователь кликает операции → тег «Мария» → отдельная категория и сумма пересчитываются). Не приписывать получателя наугад.

## Заметки
- `TRANSFER` бывает и расходом, и доходом — разводить по знаку дельты, не по метке.
- `BE1ST SMART` — всегда зачисление (кешбэк/возврат округлений), идёт с датой задним числом.
