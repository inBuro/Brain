# Рецепт дашборда трат

Цель: «презентация трат, которую интересно покрутить». Один самодостаточный файл, открывается офлайн двойным кликом, ничего никуда не отправляет.

## Состав (всё в рабочей папке вне ~/Brain)
- `index.html` — разметка + CSS + JS инлайн.
- `chart.umd.min.js` — **вендоренный** Chart.js (скачать с CDN заранее), чтобы работало без интернета через `file://`. CDN-ссылку в `<script src>` не оставлять — на `file://` может не загрузиться.
- `data.js` — данные как `window.DATA = {...};` и подключить `<script src="data.js">` (не `fetch` — на `file://` CORS-ошибка).

`window.DATA` = `{ meta, cats:[{key,label,emoji,color,type:'spend'|'income'}], tx:[{d:'YYYY-MM-DD', l:label, a:amount, c:catKey, m:'YYYY-MM'}] }`. Все агрегаты считать в JS (для переключения периода на лету).

## Дизайн (направление от ux-ui-designer, проверено)
Тёмная «ночная аналитика». Фон `#0d0f14`, поверхности `#161920`, elevated `#1e2230`, бордеры `#2a2e3e`, текст `#e8eaf2`/`#8a8fa8`. Шрифты: Inter (UI) + JetBrains Mono (цифры/KPI). Цвета категорий: дом `#f0a830`, еда `#4dc9a0`, переводы `#7b8ff5`, ATM `#e0784a`, комиссии `#5a6480`, доход `#62d16a`, кешбэк `#3f9a78`. Карточки radius 12px, контролы 8px, max-width 1280px.

## Секции (сверху вниз)
1. Шапка: бренд + сегмент-переключатель периода (Всё / по месяцам) — пересчитывает KPI, донат, топ, таблицу.
2. KPI ×4: потрачено / получено / аренда / чистый поток (цвет по знаку).
3. Бары по месяцам (доход vs расход), клик по месяцу = выбрать период.
4. Донат структуры расходов, центр = сумма; клик по сегменту/легенде = фильтр таблицы + подсветка.
5. Стек расходов по категориям по месяцам.
6. Календарный хитмап ежедневных поступлений (CSS-grid: строки=месяцы, 31 ячейка, заливка по сумме).
7. Топ-N разовых трат (горизонтальные бары, без аренды).
8. Таблица всех операций: поиск + фильтр категории + фильтр месяца; рендерить с ограничением (~600 строк) + «…и ещё N».

Chart.js: донат `doughnut cutout '70%'`; месяцы `bar` (grouped); стек `bar` stacked x+y. `Chart.defaults.color='#8a8fa8'`, font Inter. Деньги форматировать `toLocaleString('ru-RU')` + префикс `฿`.

## Интерактив (заложить)
- Переключатель периода (Всё/месяц).
- Клик по донату/легенде → фильтр таблицы + dim остальных сегментов + синхрон с дропдауном категории.
- Клик по месяцу в барах → период.
- **Режим тегирования получателя** (напр. «Мария»): т.к. имён в выписке нет — кнопка-режим, чекбоксы на строках переводов, помеченные уходят в отдельную категорию, донат/суммы пересчитываются (localStorage). По умолчанию выключен, добавлять по запросу.

## Исключение «транзитных» операций из агрегаций (флаг `x`)
Когда пользователь просит **полностью убрать** операцию из всех расчётов/визуализаций (напр. «parking» — деньги пришли и ушли, или ошибочный платёж + возврат):
- В `data.js` пометить строку(и) флагом `"x":1` — **не удалять физически**, чтобы полный набор `tx` всё ещё сходился с контрольными итогами банка.
- В **каждом** потребителе (`index.html`, отдельный `months.html` и т.п.) сразу после `const D = window.DATA;` сделать единую точку фильтрации:
  ```js
  D.txAll = D.tx;            // полный набор — для сверки/футера
  D.tx = D.txAll.filter(t=>!t.x);  // активный набор — все агрегации читают его
  ```
  Так все KPI/донат/стек/хитмап/топ/таблица/аномалии чистятся одним местом, а не правкой каждой агрегации.
- **Парность критична.** «Пришла и ушла» = обычно ДВЕ строки: списание и встречный возврат на ту же сумму (часто `PURCHASE E-CHN` ↔ `BE1ST SMART`, в один день). Они в сумме = 0 → исключив обе, не сдвинешь баланс. Проверить: `sum(a for x==1) == 0`. Убрать только одну — баланс поедет.
- Сверка: `opening + sum(всех tx) == closing` И `opening + sum(активных tx) == closing` (одно и то же, раз пара = 0). Прогнать обе.
- Статичный футер со сверкой («N списаний · M зачислений · баланс …») описывает ПОЛНЫЙ набор банка — он остаётся верен и менять его не надо.

## Проверка перед сдачей (обязательно)
Через gstack `browse` (headless): `goto file://…/index.html` → `console --errors` (должно быть пусто) → проверить число KPI/легенды/строк, отрисовку canvas → прокликать период и фильтр категории. Скриншот — для себя, пользователю не показывать (описывать текстом). Затем `open` в браузере пользователя.

**Если gstack недоступен** — headless через системный Chrome по CDP без зависимостей: `/Applications/Google Chrome.app/Contents/MacOS/Google Chrome --headless=new --disable-gpu --remote-debugging-port=PORT --allow-file-access-from-files file://…`. Node ≥22 имеет глобальный `WebSocket`. Подключаться к таргету из `GET /json` (НЕ `/json/new` — новый Chrome требует там PUT), `Runtime.enable`, ловить `Runtime.consoleAPICalled`(type:error)/`Runtime.exceptionThrown`, гонять `Runtime.evaluate` с `returnByValue:true`. Дать ~2 c на отрисовку Chart.js перед чтением DOM. Проверять: число canvas, KPI-тексты, отсутствие исключённой суммы в `document.body.innerText`, список аномалий нужного месяца.
