## Концепция

В Fadercraft нет отдельного `spacing`-слоя — density самодостаточна. Имя переменной = значение плотности, без префикса (`--6` = 24px). 1 unit = 4px.

Семантические алиасы по конкретным use-case (`--page-pad`, `--radius-md` и т.п.) остаются — они выражают смысл, а не density.

## Сверка с Typography

Все используемые значения size/line из Typography (12, 14, 16, 17, 20, 23, 24, 25, 28, 30, 34, 36, 40, 42, 48, 50, 62) покрыты density 1-в-1. Дельта = 0.

## История

- **2026-05-15 (Этап 1):** density расширена до 30 ступеней (добавлены `4,25`, `5,75`, `6,25` под нужды Typography и `15`, `20` под крупные секционные отступы). Префикс `--density-` снят, имя CSS-переменной = просто число. Параллельный слой `--space-*` удалён полностью — density самодостаточна.

- **2026-05-15 (Этап 3):** разделитель дробной части в Figma — **запятая** (`,`). Точка отвергнута: Plugin API не разрешает `.` в variable names. CSS-сторона использует дефис, как и раньше.

## Жёсткие правила

- Только из density-шкалы, никаких raw px.
- Имя переменной в Figma — число (дробная часть через запятую, ограничение Figma API); в CSS — то же число с дефисом.
- Никаких `--density-*`, `--d-*`, `--space-*`, t-shirt sizing для density.
- Семантические алиасы (radius, page-pad) — допустимы, если выражают смысл вне density.
