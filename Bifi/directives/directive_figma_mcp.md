# Директива: Figma MCP

Эта директива нужна агенту в двух режимах:
1. **Разовая установка** — когда пользователь впервые подключает Figma MCP
2. **Постоянная работа** — агент обращается к ней каждый раз, когда делает что-то с Figma через MCP

---

## Установка (один раз)

**Всегда через правку `~/.claude.json`.** Не вызывай `claude plugin install` и `claude mcp add` — CLI может отсутствовать в PATH (агент часто работает внутри VSCode-расширения, там standalone CLI нет). Ручная правка конфига универсальна и не требует отдельного бинарника.

Выполни по порядку. Без проверок условий, без альтернатив, без пропуска шагов.

**Шаг 1.** Прочитай `~/.claude.json`. Если на этот файл нет разрешения — попроси пользователя добавить read/write allow-правило на `~/.claude.json` в настройках Claude Code (`/permissions` или `.claude/settings.json`), потом продолжай.

**Шаг 2.** Найди в файле объект `mcpServers` (верхний уровень). Если его нет — создай. Добавь туда запись:

```json
"figma": {
  "type": "http",
  "url": "https://mcp.figma.com/mcp"
}
```

Если запись `figma` уже есть — не перезаписывай молча, покажи пользователю текущее значение и спроси.

**Шаг 3.** Сохрани файл. Скажи пользователю:

> Перезапусти VSCode-расширение Claude Code (или desktop-приложение), чтобы оно подцепило новый MCP-сервер. Вернись сюда и напиши «готово».

**Шаг 4.** После подтверждения попроси пользователя прислать ссылку на любой фрейм из его Figma (правый клик → Copy link to selection).

**Шаг 5.** Вызови `get_design_context` для этой ссылки. Первый вызов триггерит OAuth — в чате появится ссылка на авторизацию Figma. Скажи пользователю:

> Перейди по ссылке, в окне Figma нажми **Allow Access** в нужный workspace, вернись сюда и напиши «готово».

> **Если Figma подключали раньше — OAuth не появится.** Подключение живёт на уровне Claude Code (аккаунт/харнесс), а не проекта: токен уже есть, и `get_design_context` сразу вернёт результат. Чтобы показать экран авторизации заново или сменить аккаунт: `/mcp` → выбери `figma` → **Clear authentication** → **Authenticate**. Это команда пользователя — агент OAuth не запускает и ссылку на него не генерирует.

**Шаг 6.** После подтверждения вызови `get_design_context` ещё раз — теперь вернётся содержимое фрейма. Покажи структуру пользователю и скажи: «Figma MCP подключён, можем работать».

## Постоянная работа: справочник MCP

### Чтение (read)

| Tool | Что возвращает |
|---|---|
| `get_design_context` | иерархия слоёв, стили, компоненты, Variables выделенного фрейма |
| `get_variable_defs` | Variables и стили из выделения: цвета, типографика, spacing |
| `get_metadata` | XML-представление структуры выделения |
| `get_screenshot` | визуальный снимок выделения |
| `get_figjam` | содержимое FigJam-канваса |

### Запись на канвас (write)

Главный tool — **`use_figma`**. Общее назначение: создать, изменить, удалить, проверить любой объект. Через него:

- создание и изменение фреймов, слоёв, групп
- создание и изменение Variables (primitive и semantic)
- создание и изменение стилей (fill, text, effect)
- создание и изменение компонентов с variants
- применение и настройка Auto Layout
- связи и коннекторы на FigJam-канвасе
- создание стикеров, текста, фреймов на FigJam

### FigJam

- `generate_diagram` — строит FigJam-диаграмму из Mermaid-синтаксиса (sitemap, user flow, архитектура)

### Файлы

- `create_new_file` — создать новый Figma Design или FigJam файл
- `search_design_system` — поиск по библиотекам дизайн-системы

## Правила работы для агента

**Батчи операции.** Одна команда «почисти фрейм: переименуй слои по BEM, примени Auto Layout, убери пустые группы» — один запрос к MCP. Не дели на три отдельных.

**Идентификация объекта.** Работай либо по Figma-ссылке, либо по `node_id`, полученному из `get_design_context`. Не угадывай.

**Проверка после write.** После каждой write-операции вызови `get_design_context` или `get_screenshot` и покажи пользователю результат — не полагайся на успех команды без визуальной проверки.

**Большие файлы.** Если контекст фрейма слишком большой (много сотен слоёв) — фильтруй по секциям, запрашивай только нужную часть, не тяни всё целиком.

**Именование.** Имена Variables, слоёв, компонентов — на английском (стандарт индустрии, ожидание разработчиков). Текстовый контент внутри интерфейса — на языке продукта, обычно русском.
