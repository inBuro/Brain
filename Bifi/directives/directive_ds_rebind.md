# Директива: Переподвязка дизайн-системы к своей копии (ds_rebind)

> **Запуск:** только когда пользователь явно дал эту директиву и ссылку. Не создавай эту директиву сам и не запускай скан/пересборку по своей инициативе — ты её выполняешь, а не придумываешь.


## Задача
Ты продублировал готовый проект — Figma-файл и папку `ds/` (`foundation.md`, `components.md`, `CONTRACT.md`). Этот индекс ссылается на **исходный** файл: в нём зашиты Node ID компонентов и ссылка на оригинальный Figma. В твоей копии и ссылка, и Node ID — другие. Директива переподвязывает индекс к твоей копии: сканирует твой файл, прописывает актуальные Node ID и ссылку, **а имена и состав файлов `ds/` оставляет как есть**.

Это не пересборка ДС. Имена компонентов и токенов, разделы, правила контракта — те же. Меняются только привязки: какой Node ID и какой файл.

## Когда применять
- Ты пошёл по пути «взять готовую копию»: есть проект, надо привязать индекс к своей копии.
- Если собираешь ДС из СВОЕГО файла с нуля — это другая директива (`ds_scan`): там состав строится по твоему файлу, имена и набор будут твои.

## Предусловие
- Подключён Figma MCP (`directive_figma_mcp`).
- Копию делал через **Duplicate целого файла**, не copy-paste фреймов. При copy-paste теряется часть компонентов и связей — переподвязка не сойдётся.
- В проекте уже лежат `ds/foundation.md`, `ds/components.md`, `ds/CONTRACT.md` (пришли с копией).

## Ключевой принцип
Матч строго **по имени**. Имена при копировании стабильны (`Button` остаётся `Button`, `surface-default` остаётся `surface-default`), а Node ID и ключ файла — нет. Поэтому имя = якорь, Node ID = то, что переписываем.

---

## Фаза 1. Контекст
Один вопрос:
```
Ссылка на ТВОЮ копию — на страницу «Components» (не на корень файла).
```
Если ссылка ведёт на корень файла или на исходный (не твой) файл — переспроси. **По ссылке на корень MCP отдаёт только первую страницу** (легко получить sitemap вместо ДС) — нужна ссылка именно на страницу Components. Проверь, что `ds/`-файлы на месте. Если их нет — это не путь копии: останови и отправь на `ds_scan`.

## Фаза 2. Считать копию
Один вызов `use_figma`. По копии собрать: имена + Node ID всех `COMPONENT_SET`/`COMPONENT` на странице Components (на один уровень вглубь группирующих фреймов/секций, без рекурсии в инстансы — как в `ds_scan`); Node ID секции `Foundation`; ключ файла копии.
```js
const page = /* страница Components по ссылке */;
const found = {};               // name -> nodeId
for (const node of page.children) {
  if (node.type === "COMPONENT_SET" || node.type === "COMPONENT") found[node.name] = node.id;
  else if (node.type === "FRAME" || node.type === "SECTION")
    for (const c of node.children)
      if (c.type === "COMPONENT_SET" || c.type === "COMPONENT") found[c.name] = c.id;
}
```

## Фаза 3. Переподвязка (по именам)
1. **`components.md`** — для каждой записи найди в `found` узел с тем же именем и перепиши только строку `Node ID`. Имя, назначение, матрицу вариантов, раздел — не трогай. Состав списка не меняй: ничего не добавляй и не удаляй.
2. **Ссылка на файл** — во всех `ds/`-файлах (`foundation`, `components`, `CONTRACT`, `source` если есть) замени старый Figma URL/ключ на свой. Node ID страниц/секций (`Foundation`, `Components`) — на найденные в фазе 2.
3. **`foundation.md`** — имена и значения токенов стабильны, их не трогаем; обновляем только ссылку и, если есть, node-привязки.
4. **`CONTRACT.md`** — поменяй только строку «Figma-файл: …» и дату. Правила и таблицы «токен → место» не трогай.

## Фаза 4. Self-check
- Каждое имя из `components.md` нашло узел в копии? Перечисли **нерешённые** (для чистой копии список пустой).
- Если что-то не нашлось — копия неполная (скорее всего делалась через copy-paste, а не Duplicate). Node ID не выдумывай: пометь и предложи передублировать файл целиком.
- Совпадает число компонентов в индексе и на странице Components? Расхождение — в замечания.

## Фаза 5. Указатель в CLAUDE.md
Как в `ds_scan`: убедись, что в корневом `CLAUDE.md` проекта есть блок-указатель на `ds/CONTRACT.md`. Нет — добавь.

## Финал
```
ДС переподвязана к твоей копии.
- ds/components.md — Node ID переписаны под твой файл (по именам)
- ds/foundation.md, ds/CONTRACT.md — ссылка на твой файл
- имена и состав — без изменений
Нерешённые имена: <список или «нет»>
В Figma ничего не меняли (read-only).
```

## Правила
- Матч только по имени; Node ID никогда не угадываем.
- Состав `ds/` не меняем — это перенос привязок, не пересборка.
- Read-only по Figma: на канвасе ничего не меняем.
- Имя не нашлось → в замечания, не подставлять «похожий».
- Копия — только Duplicate целого файла, не copy-paste.
