# Стартовый набор — модуль AI Prototyper

Всё, что нужно, чтобы войти в модуль прототипирования, если ты взял его отдельно: дизайн-система проекта (индекс), агентская среда и директивы.

## Что внутри
- `ds/` — индекс дизайн-системы:
  - `foundation.md` — токены (цвета, типографика, отступы, радиусы)
  - `components.md` — каталог компонентов с вариантами и Node ID
  - `CONTRACT.md` — правила сборки: какой токен где, что можно, чего нельзя
- `CLAUDE.md` — описание агентской среды + указатель на ДС
- `directives/` — три директивы:
  - `directive_figma_mcp.md` — подключить Figma к агенту (OAuth)
  - `directive_ds_rebind.md` — переподвязать копию ДС под твой Figma-файл
  - `directive_ds_scan.md` — собрать индекс из своего Figma (если идёшь со своим продуктом)

Отдельно — ссылка на Figma-файл для дублирования (в материалах урока).

## Путь А — берёшь готовый проект (рекомендую)
1. Поставь Claude Code (расширение VS Code) и **открой эту папку** — среда уже собрана (CLAUDE.md + ds/ + directives + execution). Разворачивать из CLAUDE.md ничего не нужно.
2. Подключи Figma — запусти `directive_figma_mcp` (пройдёшь вход в Figma). Если Figma уже подключён, OAuth не появится — это нормально.
3. Открой Figma-файл из материалов → **Duplicate to your drafts** → своя копия.
4. Переподвяжи — запусти `directive_ds_rebind`, дай ссылку на **страницу Components** своей копии (не на корень файла). Агент перепишет Node ID и ссылку по именам, состав не тронет.
5. Готово: ДС + папка ds + среда + MCP. Идёшь в модуль.

## Путь Б — со своим продуктом
Шаги 1–2 те же. Дальше вместо копии — `directive_ds_scan` по своему Figma: соберёт `ds/` из твоего файла.

`ds/` в наборе — демо (CryptoVPS). На своём продукте `ds_scan` **перезапишет** его твоей системой; отдельно удалять файлы или просить агента «переделай» не нужно — просто запусти `ds_scan` на своём Figma.
Имей в виду: качество результата зависит от того, как собран твой Figma.
