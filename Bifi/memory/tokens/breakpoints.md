# Breakpoints

Источник: Figma → Variables → коллекция **Breakpoints** в `v2.4 Beefy Design System`
Статус: full (3/3 переменных, упрощено с 4 до 3 — `Desktop S` удалён 2026-05-15)
Обновлено: 2026-05-15

## Что это

Дизайн-канвасы для responsive макетов. Эти ширины — **точки, на которых дизайнер рисует** в Figma. Не обязательно совпадают 1:1 с CSS-breakpoint'ами кода (см. секцию ниже).

## Шкала

| Token | Value (px) | Когда применять |
|---|---|---|
| `Mobile` | 360 | минимальный мобильный canvas, типовой смартфон |
| `Tablet` | 960 | переход к desktop-layout; на сайте Beefy в этой точке single-column → two-column (deposit-форма становится sidebar) |
| `Desktop` | 1260 | основной desktop canvas |

## Соответствие реальному коду app.beefy.com

Проверено 2026-05-15 через визуальный обход `https://app.beefy.com/vault/aerodrome-cow-base-vvv-diem-v2-vault` на разных viewport-ширинах. Реальные транзишены вёрстки:

| Реальная точка | Что меняется | Совпадение с Figma |
|---|---|---|
| **960 px** | Mobile → desktop layout: deposit-форма из full-width single column превращается в правую sidebar-колонку (~330px) | ✅ совпадает с `Tablet` |
| **~1041 px** | Hamburger menu → полная desktop-нав (Platform / Portfolio / DAO / Resources) | ❌ нет соответствия в Figma; промежуточный header-only breakpoint |
| **1296 px** | Max content width — main выше не растёт, на 1297+ контент остаётся `1296` с возрастающими padding'ами | ❌ Figma `Desktop` = 1260, off на 36 px |

**Вывод:** Figma `Mobile=360` и `Tablet=960` точно матчат реальные дизайн-точки. `Desktop=1260` слегка off от реального max-content (1296), но это намеренная дизайн-граница — макет рисуется на 1260, а оставшееся пространство до 1296 это padding.

Hamburger-транзишен (~1041) **не дизайн-канвас**, а вторичный header-only сдвиг, не требующий отдельного брейкпоинта.

## Жёсткие правила

- Макеты рисуем строго на этих трёх ширинах: `Mobile (360)`, `Tablet (960)`, `Desktop (1260)`.
- Если макет ведёт себя по-разному в диапазоне 1041–1260 (header), это решается внутри desktop-фрейма — отдельный canvas не заводить.
- Не использовать произвольные ширины канваса (типа `768`, `1024`, `1440`).
- При проверке вёрстки в коде помни про реальный максимум `1296` (визуальные баги на широких экранах = центрирование + max-width).
