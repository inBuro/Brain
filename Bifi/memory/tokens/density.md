# Density — примитивная шкала единиц

Источник: Figma → Variables → коллекция **Density** в `v2.4 Beefy Design System`
Статус: full (27/27 переменных — `5,5` добавлен 2026-05-15 для покрытия `lineHeight-md`)
Обновлено: 2026-05-15

## Что это

Density — **примитивный нумерический слой** дизайн-системы. Это не spacing, не radius, не size — это базовая шкала чисел в пикселях, на которую ссылаются все остальные коллекции (spacing, radii, sizes, gaps, paddings, margins, иконки, line-heights и т.д.).

База — **4px grid**. Имя переменной — это step (множитель), значение — `step × 4` px. Дробные шаги (`0,25`, `0,5`, `0,75` и т.д.) дают доступ к 1/2/3/5/6 px без выхода из системы.

**Density hidden from publishing.** Все 26 переменных помечены `hiddenFromPublishing: true` — они file-internal примитивы, не экспонируются в библиотеку. Поэтому в дизайне на density обычно ссылаются не напрямую, а через семантические токены из публикуемых коллекций (Colors, Typography и т.д.), которые сами тянут значения из density.

## Naming-конвенция

**Переменная названа просто шагом**, без префикса. В Figma имя — это `{step}`, значение — `step × 4` px. Десятичный разделитель — запятая (как в Figma UI).

Примеры:
- `1` → 4 px
- `2` → 8 px
- `4` → 16 px
- `0,5` → 2 px
- `999` → 9999 px (sentinel, см. ниже)

При ссылках в дизайн-спецификациях пиши именно так: «gap = `2`», «radius = `1,5`», «width = `12`». Контекст коллекции (Density) обычно понятен из контекста — Figma показывает их как `Density/{step}` в панели переменных.

## Allowed scopes (где Figma разрешает их применять)

Каждая переменная имеет одинаковый набор scopes — то, к чему её можно биндить:

- `CORNER_RADIUS` — border-radius
- `WIDTH_HEIGHT` — ширина/высота элементов
- `GAP` — auto-layout gap
- `STROKE_FLOAT` — толщина бордера/обводки
- `FONT_SIZE` — размер шрифта
- `LINE_HEIGHT` — line-height
- `LETTER_SPACING` — letter-spacing
- `PARAGRAPH_SPACING` — отступ между параграфами

То есть density умышленно может быть привязан **ко всему числовому**: расстояние, размер, радиус, толщина, типографика. Это и есть «универсальный нумерический слой».

Чего нет в scope: opacity, percentage, scale (не предполагается использовать density для этих случаев).

## Особенности шкалы

- **Базовый grid 4px.** Большинство значений — кратные 4 (4, 8, 12, 16, 20, 24, 28, 32, 36, 40, 48).
- **Дробные шаги для 2px-выравнивания** (`0,5; 1,5; 2,5; 3,5; 4,5; 6,5; 7,5; 10,5; 12,5; 15,5`) — для случаев, когда чистый 4px-grid слишком груб.
- **Совсем мелкие шаги** (`0,25; 0,75; 1,25`) — для 1px / 3px / 5px hairlines.
- **Пробелы в шкале**: между `12,5` (50) и `15,5` (62) есть скачок без 13/14/15. Между `8` (32) и `9` (36) пропущен `8,5`. Это намеренно — большие значения нужны реже, и шкала прорежена. Если нужно промежуточное значение — это сигнал, что задача либо требует исключения, либо неверно поставлена.
- **`999` sentinel.** Используется как «pill / full radius / max-width / spacer-grow». Не добавлять обычный 9999 hardcoded — ссылаться на `999`.

## Где применять

Density — primitive layer. Можно ссылаться напрямую (`gap = 4`), но чаще density используется как **источник для семантических токенов** в других коллекциях (spacing, radii, sizes). В дизайне:

- **gaps между элементами**: `1` / `2` / `3` / `4`
- **paddings внутри компонентов**: `2` / `3` / `4` / `6`
- **section spacing**: `6` / `8` / `10` / `12`
- **border radius**: `0,5` / `1` / `2` / `3` / `999` (full)
- **icon sizes**: `4` (16) / `5` (20) / `6` (24)
- **avatar/control sizes**: `6` (24) / `8` (32) / `10` (40) / `12` (48)
- **hairlines / borders**: `0,25` (1px)

## Жёсткие правила

- **Никаких raw px в дизайне.** Всё, что выглядит как pixel value (spacing, radius, size, gap, padding) — должно ссылаться на density.
- **Не вводить новые ступени без обсуждения.** Если значения из шкалы не хватает — сначала вопрос «а почему», потом расширение коллекции.
- **`999` = full / max**, не использовать буквальный 9999 в макетах.
- **Дробные ступени — только когда 4px-grid недостаточен.** Default — целочисленный шаг.
