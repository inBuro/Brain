
# Raw — Inbox

Глобальный инбокс. Сюда попадают входящие данные «как есть» до того, как их разнесут по проектам.

## Что сюда класть

- Заметки из головы, которые ещё не привязаны к проекту.
- Ссылки, цитаты, фрагменты переписок, идеи.
- Голосовые расшифровки.
- Любой сырой материал, который потенциально станет частью проекта.

## Что **не** класть

- Большие файлы (PDF, изображения, аудио) — клади в `<Project>/raw/assets/` напрямую, если проект уже определён, или в `_inbox-assets/` (создать при необходимости).
- Готовые обработанные страницы wiki — они идут сразу в `<Project>/wiki/`.

## Жизненный цикл записи

```
[ ] входящее      ← новая запись со статусом-чекбоксом
[~] в обработке   ← LLM или ты начали разбор
[x] разнесено     ← ушло в проект, можно архивировать или удалять
```

В записи указывай дату и (если знаешь) целевой проект.

## Формат записи

```markdown
### [YYYY-MM-DD] <короткий заголовок>
- status: [ ] | [~] | [x]
- target: <Project Name> | unknown
- type: idea | source | note | link | quote

<тело записи — свободная форма>

---
```

После переноса в проект: пометить `[x]`, добавить ссылку на созданную страницу wiki, в `log.md` записать `process`.

Раз в N дней (или по запросу) — чистка: удалить `[x]`-записи старше недели.
